- **TCP 拥塞控制算法**：
    
    - **核心过程**：必须掌握慢开始（指数增长）、拥塞避免（加性增）、快重传（收到3个冗余ACK）和快恢复 1111。
        
    - **关键公式**：发送窗口的上限值 = $min[rwnd, cwnd]$ 2222。
        
    - **异常处理**：出现网络超时时，ssthresh 设为当前 cwnd 的一半，cwnd 重置为 1，重新开始慢开始 3333。 
        
- **IPv4 地址规划与计算**：
    
    - **子网划分与 CIDR**：给定 IP 地址块进行子网划分或路由聚合，计算网络号、子网掩码及可用主机范围 4444。
        
    - **IP 数据报分片**：根据 $MTU$ 计算分片的数据长度、偏移量和标志位（注意：只有最后一片的有效载荷不是 8B 的倍数） 5555。
        
- **路由算法（Dijkstra）**：
    
    - **OSPF 压轴题**：通常要求使用迪杰斯特拉算法计算最短路径，并写出最终的路由表或转发表格式 666666666。
        
- **UDP 校验和计算**：二进制反码求和运算再取反的过程 7777。

### 二、 必背流程与简答题（6-8分/题）

- **TCP 三次握手与四次挥手**：
    
    - 要求：不仅要背过图，还要能准确写出每个阶段的状态（如 `SYN-SENT`, `ESTABLISHED`, `TIME-WAIT` 等）以及序号 $seq$ 和确认号 $ack$ 的变化 888888888。
        
- **地址解析协议 ARP**：
    
    - 掌握主机 A 向不同子网主机 B 发送报文的完整过程：判断子网 -> 获取网关 MAC -> 路由器转发 -> 获取目的主机 MAC 9999。
        
- **NAT 网络地址转换**：
    
    - 理解私有 IP 如何通过 NAT 路由器的全球 IP 和不同端口号访问公网，掌握转换表表项的生成过程 10101010。
        
- **DHCP 交互过程**：四个阶段：发现（Discovery）、提供（Offer）、请求（Request）、确认（ACK），均为广播形式 11。
    
- **DNS 解析流程**：递归查询与迭代查询的区别；掌握 MX 记录的解析过程（先查邮件服务器名，再查其 IP） 12121212。
    
- **网络安全方案**：发送方使用 MD5 摘要、RSA 私钥签名、AES 会话密钥加密的完整组合流程 13。
