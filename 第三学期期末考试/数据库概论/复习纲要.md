数据库概论（Database System Concepts）和概率论完全不同。概率论靠“算”，数据库靠“逻辑”和“记忆”。

这门课在期末考试中，考点**极其固定**。无论你用的是萨师煊（人教版）还是外国教材（Silberschatz），核心考点都逃不过下面这**三大板块**。

我结合你之前表现出的**Java后端**和**索引（B+树）**背景，为你划出重点：

---

### 第一板块：SQL与关系代数 (分值最重，约占40%)

这是拿分的核心，也是你作为后端预备役最熟悉的部分，但考试和写代码不一样，有**特定格式**。

1. **SQL 查询语句 (必考大题)**
    
    - **核心考点：** 多表连接 (`JOIN`)、分组筛选 (`GROUP BY` + `HAVING`)、嵌套子查询 (`IN`, `EXISTS`)。
        
    - **经典陷阱：** `WHERE` 和 `HAVING` 的区别（`WHERE` 筛行，`HAVING` 筛组）。
        
    - **除法问题：** “查询选修了**所有**课程的学生”。这是最难的SQL题，通常用 `NOT EXISTS` 双重嵌套来写。
        
2. **关系代数 (Relational Algebra)**
    
    - 考试会让你用希腊字母 ($\sigma, \pi, \bowtie, \div$) 写查询。
        
    - **必考符号：**
        
        - **选择 ($\sigma$)**：对应 SQL 的 `WHERE`。
            
        - **投影 ($\pi$)**：对应 SQL 的 `SELECT`。
            
        - **连接 ($\bowtie$)**：自然连接，最常用。
            
        - **除法 ($\div$)**：对应“查询完成**所有**...”这类问题。**一定要背除法的定义和写法！**

### 第二板块：数据库设计理论 (逻辑性最强，约占30%)

这部分比较抽象，是很多人的丢分点，但其实全是套路。

1. **范式 (Normalization)**
    
    - **核心技能：** 给你一个关系模式 $R$ 和函数依赖集 $F$，让你：
        
        1. **求候选码 (Candidate Key)：** 这是所有题目的起点，必须熟练掌握“闭包计算算法”。
            
        2. **判断范式等级：**
            
            - **1NF：** 属性不可分（默认都满足）。
                
            - **2NF：** 消除**非主属性**对码的**部分**函数依赖（联合主键常见）。
                
            - **3NF：** 消除**非主属性**对码的**传递**函数依赖。
                
            - **BCNF：** **所有**决定因素都是码（消除主属性对码的部分/传递依赖）。
                
        3. **模式分解：** 把一个不好的表拆成几个符合3NF的表（要在保持函数依赖、无损连接的前提下）。
            
2. **E-R 图 (Entity-Relationship Diagram)**
    
    - **画图：** 实体（矩形）、属性（椭圆）、关系（菱形）。不要漏掉主键下划线。
        
    - **转换成关系模式 (重中之重)：** 把画好的图转成表。
        
        - **1:1 关系：** 归到任意一端。
            
        - **1:n 关系：** 归到 n 的那一端（把“1”的主键拿过来做外键）。
            
        - **m:n 关系：** **必须单独建一张表**（包含两边的主键 + 关系本身的属性）。

### 第三板块：系统原理 (背诵+理解，约占30%)

你之前对**索引**和**并发**感兴趣，这部分对你来说应该比较亲切。

1. **事务 (Transaction)**
    
    - **ACID 四大特性：** 原子性、一致性、隔离性、持久性（必考填空/简答）。
        
    - **并发控制：**
        
        - **封锁协议：** 一级（防丢失修改）、二级（防脏读）、三级（防不可重复读）封锁协议的区别。
            
        - **死锁：** 产生原因和解决办法。
            
        - **可串行化调度：** 判断一个并发调度是否正确（通常看它是冲突可串行化的）。
            
2. **存储与索引 (你之前问过的B+树)**
    
    - 虽然你懂B+树原理，但考试通常考：
        
        - **计算题：** 给定数据量和块大小，算树的高度或I/O次数。
            
        - **画图题：** 插入一系列数字，画出B+树分裂的过程（节点满了怎么拆）。
            
    - **聚簇索引 vs 非聚簇索引** 的区别。
        
3. **恢复 (Recovery)**
    
    - **日志 (Log)：** Undo 和 Redo 的概念。
        
    - **故障恢复：** 检查点 (Checkpoint) 机制。给你一个日志序列，让你判断系统重启后哪些事务要 Redo，哪些要 Undo。

### ⚠️ 避坑指南

1. **SQL 别写成代码：** 考试是手写，标点符号要规范，不要写简写。
    
2. **关系代数除法：** 如果没时间复习，把“查询选修了全部课程的学生”这个关系代数公式背下来，改改变量名就能用。
    
3. **范式判断：** 一定要先算对**候选码**！候选码算错了，后面判断2NF/3NF全错。
